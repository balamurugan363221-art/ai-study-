# ⚡ AI Study Buddy

An AI-powered academic assistant built with **Flask + SQLite**, featuring secure authentication, an AI chat interface, and a study progress tracker.

---

## 🚀 Run Locally

### 1 – Clone / download the project

```bash
git clone <your-repo-url>
cd ai-study-buddy
```

### 2 – Create a virtual environment

```bash
python -m venv venv

# macOS / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 3 – Install dependencies

```bash
pip install -r requirements.txt
```

### 4 – Configure environment variables

```bash
cp .env.example .env
```

Edit `.env`:

```
SECRET_KEY=any-long-random-string
OPENROUTER_API_KEY=   # optional – get free key at https://openrouter.ai
HUGGINGFACE_API_KEY=  # optional – get free key at https://huggingface.co/settings/tokens
```

> The app works without an AI key – it shows a setup prompt in the chat instead.

### 5 – Initialise the database & run

```bash
python -c "from app import init_db; init_db()"
python app.py
```

Open **http://localhost:5000** in your browser.

|---|---|---|
| **OpenRouter** | Sign up at https://openrouter.ai → API Keys → Create | $1 free credit (plenty for testing) |

Add your key to `.env` and restart the server – that's it.

## ☁️ Deploy to Render (Free)

1. Push the project to a GitHub repository.
2. Go to https://render.com → **New Web Service** → connect your repo.
3. Render auto-detects `render.yaml` – click **Apply**.
4. In the Render dashboard, set your env vars:
   - `OPENROUTER_API_KEY` (or `HUGGINGFACE_API_KEY`)
   - `SECRET_KEY` is auto-generated by `render.yaml`.
5. Hit **Deploy** – your app will be live in ~2 minutes.

## 🔐 Security Features

- Passwords hashed with **bcrypt** (never stored in plain text)
- All database queries use **parameterised statements** (no SQL injection)
- API keys stay **server-side only** (never sent to the browser)
- Session-based auth with **login_required** decorator on all protected routes
- Input validation and length limits on all API endpoints

## 📋 Database Schema

```sql
-- Users
CREATE TABLE users (
    id       INTEGER PRIMARY KEY AUTOINCREMENT,
    username TEXT NOT NULL UNIQUE,
    email    TEXT NOT NULL UNIQUE,
    password TEXT NOT NULL          -- bcrypt hash
);

-- Chat history (per user)
CREATE TABLE chat_history (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id   INTEGER NOT NULL REFERENCES users(id),
    question  TEXT NOT NULL,
    answer    TEXT NOT NULL,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Study progress (per user)
CREATE TABLE study_progress (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL REFERENCES users(id),
    subject TEXT NOT NULL,
    hours   REAL NOT NULL,
    date    DATE DEFAULT (DATE('now'))
);
```

## 💡 Tips

- Switch between **dark / light mode** with the Theme button in the sidebar.
- Press **Enter** to send a chat message (Shift+Enter for a new line).
- Study hours are grouped by subject so you can track each one independently.
- Chat history shows your last 20 questions – only visible to you.
